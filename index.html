P1: Data Types, Operators, & Ternary

Simple Explanation:

This covers the absolute basics: how to store data using **Data Types** and how to use the **Ternary Operator** (a one-line `if-else` shortcut) for simple conditional logic.

Detailed Breakdown:

  • **Data Types**: You must know the difference between int (whole numbers), double (decimals), boolean (true/false), and String (text).
  • **Ternary**: Structure is (Condition ? Value_If_True : Value_If_False). It's concise but limit nesting.
  • **Classes**: Defines the blueprint for objects, demonstrated by the Student class structure.

Java Code:


package prac1;

public class Main {
    public static void main(String[] args) {
        // Data Types Demonstration
        int age = 25;
        String name = "Alice";
        double price = 19.99;

        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
        
        // Ternary Operator Example: Check Even/Odd
        int number = 10;
        String result = (number % 2 == 0) ? "Even" : "Odd";
        System.out.println("Number " + number + " is " + result);

        // Nested Ternary
        int x = 5;
        int z = (x > 0) ? ((x < 10) ? 1 : 2) : 3;
        System.out.println("Value of z: " + z);
    }
}

class Student {
    String name = "Tom";
    int id = 202503054;
    public void displayInfo() {
        System.out.println(name + " " + id);
    }
}

Console Output:

Name: Alice
Age: 25
Number 10 is Even
Value of z: 1

P2: Control Flow (If-Else, Loops, Switch)

Simple Explanation:

Control Flow dictates program order. Use **If/Else** for decisions and **Loops** (`for`/`while`) for repeating a block of code.

Detailed Breakdown:

  • **If/Else**: Used for branching logic based on complex conditions (e.g., scoring grades).
  • **For Loop**: Use when you **know** the number of repetitions (fixed iteration).
  • **Switch**: A clean way to handle one variable checked against many **fixed values**.

Java Code:


package p2;

public class Main {
    public static void main(String[] args) {
        // 1. If-Else Control Flow
        int score = 75;
        System.out.print("Grade Check: ");
        if (score >= 90) {
            System.out.println("A");
        } else if (score >= 70) {
            System.out.println("B"); 
        } else {
            System.out.println("C");
        }

        // 2. For Loop
        System.out.print("For Loop (1-3): ");
        for (int i = 1; i <= 3; i++) {
            System.out.print(i + " ");
        }

        // 3. Switch Statement
        int day = 3;
        String dayName;
        switch (day) {
            case 1: dayName = "Monday"; break;
            case 3: dayName = "Wednesday"; break;
            default: dayName = "Weekend"; break;
        }
        System.out.println("\nSelected Day: " + dayName);
    }
}

Console Output:

Grade Check: B
For Loop (1-3): 1 2 3 
Selected Day: Wednesday

P3: Polymorphism (Overloading, Overriding)

Simple Explanation:

Polymorphism means "many forms." **Overloading** happens in the same class (different inputs). **Overriding** happens across classes (child redefines parent method).

Detailed Breakdown:

  • **Overloading (Compile-Time)**: Java chooses the correct method based on the parameters at compilation.
  • **Overriding (Runtime)**: Java chooses the method based on the actual *object* type (e.g., SBI) at runtime. Use @Override.
  • **Example**: The Bank parent class is used as a generic reference, but the specific bank's rate is returned.

Java Code:


package p3;

// Method Overloading
class Volume {
    public float volume(float l) { return l*l*l; } // Cube (1 arg)
    public double volume(float r, float h) { return Math.PI*r*r*h; } // Cylinder (2 args)
}

// Runtime Polymorphism (Overriding)
class Bank { float getRateOfInterest() { return 0; } }
class SBI extends Bank { @Override float getRateOfInterest() { return 8.5f; } }
class HDFC extends Bank { @Override float getRateOfInterest() { return 5.2f; } }

public class Main {
    public static void main(String[] args) {
        Volume v = new Volume();
        System.out.println("Cube: " + v.volume(5));
        System.out.println("Cylinder: " + v.volume(5,5));

        Bank b; // Parent reference
        b = new SBI(); 
        System.out.println("SBI Rate: " + b.getRateOfInterest());
        b = new HDFC();
        System.out.println("HDFC Rate: " + b.getRateOfInterest());
    }
}

Console Output:

Cube: 125.0
Cylinder: 392.69908169872417
SBI Rate: 8.5
HDFC Rate: 5.2

P4: Abstract Class & Method

Simple Explanation:

An **abstract class** is a template that cannot be instantiated. An **abstract method** has no body and **forces** any non-abstract subclass to implement it.

Detailed Breakdown:

  • **`abstract` method**: Used to define a contract. The method signature exists, but the implementation is left blank (no `{}`).
  • **`abstract` class**: Used to group related classes (e.g., Sphere and HemiSphere) under a common required method (calVolume).
  • **Rule**: You cannot create an object of an abstract class.

Java Code:


package p04;

abstract class Volume {
    abstract void calVolume(); // Abstract: no body, must be implemented
    void printIntro() { System.out.println("--- Volume Calculation Started ---"); }
}
class Sphere extends Volume {
    double r;
    Sphere(double r) { this.r = r; }
    @Override 
    void calVolume() { // Forced implementation for Sphere
        double result = (4.0/3.0)*Math.PI*r*r*r;
        System.out.println("Volume of Sphere: " + result);
    }
}
class HemiSphere extends Volume {
    double r;
    HemiSphere(double r) { this.r = r; }
    @Override 
    void calVolume() { // Forced implementation for HemiSphere
        double result = (2.0/3)*Math.PI*r*r*r;
        System.out.println("Volume of Hemi-Sphere: " + result);
    }
}

public class Main {
    public static void main(String[] args) {
        Sphere s = new Sphere(2);
        s.printIntro(); 
        s.calVolume();

        new HemiSphere(2).calVolume();
    }
}

Console Output:

--- Volume Calculation Started ---
Volume of Sphere: 33.510321638291124
Volume of Hemi-Sphere: 16.755160819145562

P5: Access Specifiers & Inner Class

Simple Explanation:

**Access Specifiers** (`private`, `public`) control which parts of the code can see your variables/methods. **Inner Classes** are classes nested inside others for better structure.

Detailed Breakdown:

  • **`private`**: Strongest restriction. Data can only be accessed within its own class (demonstrated by Person.age).
  • **Inner Class (Non-Static)**: Tied to an outer object and can access **all** members (even private ones) of that outer object.
  • **Static Nested Class**: Independent of the outer object but grouped logically. Can **only** access **static** members of the outer class.

Java Code:


package p5;

// Access Specifiers Demo
class Person {
    public String name = "Alice";
    private int age = 30; // Hidden data
    
    public void accessPrivate() { showAge(); } // Accesses private helper method
    private void showAge() { System.out.println("Age: " + age); }
}

// Inner Class Demo
public class Outer {
    String message = "Hello from Outer class!";
    private static String staticMessage = "Static Outer Message";
    
    // A) Member Inner Class (Accesses 'message')
    class Inner {
        void display() { System.out.println("Inner class says: " + message); }
    }
    
    // B) Static Nested Class (Accesses 'staticMessage')
    static class StaticNested {
        void display() { System.out.println(staticMessage); }
    }

    public static void main(String[] args) {
        // Inner Class Call
        Outer o = new Outer();
        Outer.Inner i = o.new Inner();
        i.display();
        
        // Static Nested Class Call
        Outer.StaticNested nested = new Outer.StaticNested();
        nested.display();
        
        // Access Private Demo
        Person p = new Person();
        p.accessPrivate();
    }
}

Console Output:

Inner class says: Hello from Outer class!
Static Outer Message
Age: 30

P6: Exception Handling

Simple Explanation:

Exception handling prevents crashes. You **try** risky code, **catch** the error, and the **finally** block runs no matter what.

Detailed Breakdown:

  • **`try`**: Block containing code that might throw a runtime exception (e.g., division by zero).
  • **`catch`**: Executes if the specified exception type is thrown. It allows the program to continue.
  • **`finally`**: Essential for cleanup operations (closing file streams, database connections). **It always executes**.
  • **User-defined**: We create a custom error by extending the Exception class and use throw new MyException().

Java Code:


package p6;

class VoteException extends Exception {
    public VoteException(String s) { super(s); }
}

public class Main {
    public static void main(String[] args) {
        // try-catch-finally
        try {
            int a = 100/0; // Causes ArithmeticException
        } catch (Exception e) {
            System.out.println("Caught Error: " + e.getMessage());
        } finally {
            System.out.println("FINALLY: Cleanup code executed.");
        }

        // Multiple exceptions example
        try { 
            String str = null; 
            System.out.println(str.length()); // NullPointerException
        }
        catch (Exception e) { 
            System.out.println("Caught another error: " + e.getClass().getSimpleName()); 
        }

        // User-defined Exception
        try {
            if (15 < 18) throw new VoteException("You r not eligible...");
        } catch (VoteException e) {
            System.out.println("\nCustom Error: " + e.getMessage());
        }
    }
}

Console Output:

Caught Error: / by zero
FINALLY: Cleanup code executed.
Caught another error: NullPointerException

Custom Error: You r not eligible...

P7: Multithreading (Odd & Even)

Simple Explanation:

Multithreading allows multiple parts of the program to run **concurrently** (at the same time). You create threads by **Extending `Thread`** or **Implementing `Runnable`**.

Detailed Breakdown:

  • **Key Method**: You **MUST** call t.start(). This creates a new execution path. Calling t.run() just executes the code sequentially.
  • **Interleaving**: We use Thread.sleep() to pause the threads at different rates (0.5s vs 1.0s), demonstrating how the operating system mixes their outputs.
  • **Runnable**: Preferred because it allows your class to extend another class later (Java only allows single inheritance).

Java Code:


package p7;

class ThreadOne extends Thread { // Method 1: Extend Thread (Odds)
    public void run() {
        for (int i = 1; i <= 10; i += 2) {
            System.out.println("ThreadOne - Count (Odd): " + i);
            try { Thread.sleep(500); } 
            catch (InterruptedException e) { e.printStackTrace(); }
        }
    }
}

class ThreadTwo implements Runnable { // Method 2: Implement Runnable (Evens)
    public void run() {
        for (int i = 2; i <= 10; i += 2) {
            System.out.println("ThreadTwo - Count (Even): " + i);
            try { Thread.sleep(1000); }
            catch (InterruptedException e) { e.printStackTrace(); }
        }
    }
}

public class Main {
    public static void main(String[] args) {
        ThreadOne t1 = new ThreadOne();
        Thread t2 = new Thread(new ThreadTwo()); // Wrap Runnable
        t1.start(); // Start threads
        t2.start(); 
        System.out.println("Main thread finished setting up.");
    }
}

ThreadOne - Count (Odd): 5
ThreadTwo - Count (Even): 4
ThreadOne - Count (Odd): 7
